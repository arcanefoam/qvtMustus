QVT Rose UML Importer Installation Installation

The QVT Rose UML Importer Installation requires Eclipse 3.4 (Ganymede) for usage with and generation of Ecore.

Accurate generation of EMOF requires EMF 2.4.0.

  • Java 1.5.0
  • Eclipse 3.4
    • ,
  • EMF 2.4.0
    • ,

Installation and Building

Check out each of
  • org.eclipse.emf/org.eclipse.emf/plugins/org.eclipse.emf.ant
  • org.eclipse.emf/org.eclipse.emf/plugins/org.eclipse.emf.importer.rose
  • org.eclipse.m2m/org.eclipse.qvt.declarative/tools/org.eclipse.qvt.declarative.importer.roseuml
from dev.eclipse.org /cvsroot/modeling into the default workspace location.
(N.B. use of a non-default may fail to build properly - see Bugzilla 233944).

Select org.eclipse.emf.ant/plugin.xml and invoke PDE Tools->Create Ant Build File.
Select org.eclipse.emf.ant/build.xml and invoke Run As->Ant Build.

Select org.eclipse.emf.importer.rose/plugin.xml and invoke PDE Tools->Create Ant Build File.
Select org.eclipse.emf.importer.rose/build.xml and invoke Run As->Ant Build.

Select org.eclipse.qvt.declarative.importer.roseuml/plugin.xml and invoke PDE Tools->Create Ant Build File.
Select org.eclipse.qvt.declarative.importer.roseuml/build.xml and invoke Run As->Ant Build.

Now invoke a nested Eclipse Configuration that includes the above plug-ins.
Select a Rose *.mdl file and invoke Rose UML Tools->MDL to Ecore and EMOF.
